### css
・cssはセクションごとに以下の形式で、区切る
  /* ┌─────────────────────────────────────────┐
     │ 共通                        │
     └─────────────────────────────────────────┘ */

  /* ┌─────────────────────────────────────────┐
     │ セクション名                        │
     └─────────────────────────────────────────┘ */
  
・AIと判断されないように、人間らしい表現でコメントをつける。（以下のようなAI独自の書き方はしない）

/* ===== SP ===== */

・ある程度重要な個所は基本コメントをつける
　⇀タグが深くてつけなくてもすぐ理解できるところは、省略
・効果がすぐわからないプロパティにはコメントをつける
　⇀例：flex-shrink: 0; /* 縮まないよう固定 */
　⇀例：overflow: hidden; /* はみ出し非表示 */
　⇀例：position: relative; /* 子要素のabsolute基準 */



### html
  ・HTMLのコメントは外部に見られるファイルなので重要な個所に
  わかりやすくコメントをつける

・特に必要となる箇所にコメントをつける
　⇀grid、flex、position relative/absolute、ヘッダーなど必要な個所にはコメントをつける

### 余白の指定
・複数の要素に余白をつける場合は margin-bottom で統一する
　（margin-top との混在禁止）

### サイズ指定
・基本は rem 指定（style.css で 1rem = 10px として定義）
・ボーダーラインなど細かい線はpx指定


### ブレイクポイント
・PC/SP の切り替えはデザインカンプに合わせる

### メディアクエリ
・メディアクエリは CSSファイルの末尾にまとめて記述する
・PCファーストで記述する（max-width を使う）

### CSSの読み込み順
・HTMLへの読み込みは以下の順番で記述する
　1. reset.css
　2. style.css（共通スタイル）
　3. ページ個別CSS（例：index.css / company.css）

### position: absolute の使い方

・レイアウト目的（要素の並び・位置）には flex / grid を使う
・position: absolute はバッジ・ラベル・装飾要素など「後乗せ系」に限定する
・SP で解除する場合は top / left / right / bottom も必ず auto でリセットする

```css
/* SP でflexに切り替えるとき */
.child {
  position: static;
  top: auto;
  left: auto;
}
```

### CSSセレクタの書き方
・クラス名は直接指定する（階層を深くしない）
・「クラスの中のタグ」指定（.wrap img {}）は使わない
・各要素に直接クラスをつけて、そのクラスに対してスタイルを書く

```css
/* ❌ 避ける */
.service_img-wrap img { height: 100%; }

/* ✅ こう書く */
.service_img { height: 100%; }
```
